# Agent 真实场景评测

首批场景固定在 `scenarios.json`，数量维持在 20～50 个。新增场景应先来自人工测试或真实失败记录，并同时维护参考答案、必须行为、禁止行为和 transcript 审核备注。

默认评测使用本地 fixture、固定网页快照或录制结果；实时联网检查必须单独运行，不能与稳定回归分数混合。

评分器位于 `app/evals/runner.py`：确定性行为检查负责工具、恢复、终止和边界，答案关键词只提供基础内容覆盖。任何模型评分器都必须独立版本化并先经过人工校准。

失败 transcript 会写入评测输出目录的 `failures/`。维护人每周抽查失败记录；当至少 20 个场景的通过率达到 95% 时，报告标记为饱和，需要增加难度或替换失去区分度的场景。

维护责任默认归 Agent 主线负责人；修改运行时、工具路由、公开事件或终止/guidance 行为时，必须同步更新相应场景。
